Key issues found and hardened

populate_config_cache had unbounded IPC growth and did not reliably detect a final unterminated partial field when EOF arrived without a trailing NUL. I added source-size, field-size, record-count, and protocol-byte caps, plus explicit partial-field detection.
The parser sandbox used one shared Lua environment for require()d files; Hyprland documents require() as separate file scope, so I changed the sandbox to execute required modules in fresh environments while sharing only the inert hl capture surface.
The sandbox now fails closed on NULs, non-finite numbers, excessive nesting, massive tables, oversized strings, oversized output, and output close/write failures.
write_value_to_file did not check lock-file open failure, value-transfer write failure, scratch stat failure, or stale source changes between mutation and atomic commit. Those paths are now explicit.
Atomic writes now use same-directory temp files, checked sync -f before mv -T, permission preservation, cleanup registration, and a source signature check to reject stale writes if the config changed mid-edit.
The mutator now handles hl.config({ ... }) and hl.config { ... }, bracket string keys, long-bracket strings with delimiter escalation, escaped string keys via Lua’s own literal parser, and refuses computed expressions instead of corrupting custom logic.
TTY handling now reads from /dev/tty, restores stty on HUP/INT/QUIT/TERM/EXIT, avoids emitting alternate-screen cleanup during --help, and disables IXON to avoid Ctrl-S terminal freezes.
Integer and float bounds are validated at registration, float math rejects non-finite values, and integer arithmetic avoids unsafe oversized Bash arithmetic input.
